{% extends "modal/base.html" %} {% block title %}Time Settings{% endblock %} {% block current %}time{% endblock %} {% block body %} This box allows you to adjust which mode the Crisees frontend runs in - either real-time or historic. To switch between modes, simply adjust the slider for the mode you wish to switch to; you will notice that the top of the box then turns green, indicating the currently selected mode. Note that switching to historic mode whilst the event is sourcing does not stop the system from gathering new information.

Real-Time

Real-time mode queries the Crisees API in intervals to retrieve the latest information from the currently active social media streams for this event. Use the slider below to adjust the refresh interval. Acceptable times range from 5 seconds to 1 minute between refreshes.

Historic

Historic mode allows you to wind back the clock and query the Crisees API for previously indexed information between the two points in time that you can select using the slider below. The information from this period is then loaded into the Crisees interface.
